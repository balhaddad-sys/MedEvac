سياسة الخصوصية

آخر تحديث: مارس 2026

تطبيق MedEvac هو نظام تسجيل مرضى مخصص لمستشفى مبارك الكبير. نحن نأخذ خصوصية بيانات المرضى على محمل الجد.

البيانات التي نجمعها

يقوم التطبيق بتخزين البيانات التالية لأغراض إدارة المرضى حصراً:

  • اسم المريض الكامل
  • رقم البطاقة المدنية
  • الجنسية
  • موقع الجناح والسرير
  • درجة الخطورة (كود التصنيف)
  • ملاحظات طبية

كيفية استخدام البيانات

تُستخدم البيانات حصرياً لأغراض:

  • تسجيل وتتبع المرضى داخل أقسام المستشفى
  • عرض إحصائيات الأقسام والفرز حسب درجة الخطورة
  • تسهيل التواصل بين الكوادر الطبية

تخزين البيانات

يتم تخزين البيانات في قاعدة بيانات Firebase Realtime Database المستضافة على خوادم Google Cloud. البيانات محمية بقواعد أمان قاعدة البيانات ونظام المصادقة عبر رمز PIN.

مشاركة البيانات

لا نقوم ببيع أو مشاركة أو نقل بيانات المرضى لأي طرف ثالث. البيانات متاحة فقط لموظفي المستشفى المصرح لهم.

خدمات الطرف الثالث

  • Google Firebase: لتخزين ومزامنة البيانات في الوقت الفعلي
  • Anthropic Claude AI: لاستيراد بيانات المرضى من الصور (اختياري) - لا يتم تخزين الصور

أمان البيانات

نحمي البيانات من خلال:

  • نظام مصادقة برمز PIN لكل قسم
  • فصل البيانات حسب الأقسام والجنس
  • اتصالات مشفرة عبر HTTPS
  • قواعد أمان قاعدة البيانات في Firebase

حقوق المستخدم

يحق لإدارة المستشفى طلب حذف أو تعديل أي بيانات مخزنة في النظام عبر التواصل معنا.

Privacy Policy

Last updated: March 2026

MedEvac is a patient registry system designed for Mubarak Al-Kabeer Hospital. We take patient data privacy seriously.

Data We Collect

The app stores the following data solely for patient management purposes:

  • Patient full name
  • Civil ID number
  • Nationality
  • Ward and bed location
  • Severity code (triage classification)
  • Medical notes

How We Use Data

Data is used exclusively for:

  • Registering and tracking patients within hospital departments
  • Displaying department statistics and triage-based sorting
  • Facilitating communication among medical staff

Data Storage

Data is stored in Firebase Realtime Database hosted on Google Cloud servers, protected by database security rules and PIN-based authentication.

Data Sharing

We do not sell, share, or transfer patient data to any third party. Data is only accessible to authorized hospital staff.

Third-Party Services

  • Google Firebase: Real-time data storage and synchronization
  • Anthropic Claude AI: Optional image-based patient data import (images are not stored)

Data Security

  • PIN-based authentication per department
  • Data separation by department and gender
  • Encrypted connections via HTTPS
  • Firebase database security rules

User Rights

Hospital administration may request deletion or modification of any stored data by contacting us.
